# \<NotifyModal />

## Component props
<AssignableProps
  data={[
    {name: 'close', value: '() => void', required: true, definition: 'Function that executes whenever a user closes the modal'},
    {name: 'closeLabel', value: 'string', required: true, definition: 'Label for the close button'},
    {name: 'message', value: 'string | []string', required: true, definition: 'Modal message'},
    {name: 'modalType', value: '"success" | "warning" | "error"', required: true, definition: 'Specify the modal type'},
    {name: 'title', value: 'string', required: true, definition: 'Modal title'},
    {name: 'callback', value: '() => void', required: false, definition: 'Execute the function after the set timeout'},
    {name: 'timeout', value: 'number', required: false, definition: 'set a timeout for the modal, if no callback is provided the close function will be used instead'},
  ]}
/>

### Separate message in paragraphs
If you would like to separate the message into different paragraphs, use an array of strings.

<MultiLineModal/>

&NewLine;

```tsx
<NotifyModal
  title={'Multiple messages'}
  message={['first string', 'this is a second text', 'Each string is wrapped in a paragraph']}
  close={() => setModalVis(false)}
  closeLabel={'close'}
  modalType={'success'}
/>
```

### Timeout
Execute a `callback` function after the set `timeout`. The `callback` function will also be executed if a
user closes the modal before the timer runs out. \
Note that `callback` functions will be ignored if you don't set a `timeout`.

<CallbackModal/>

&NewLine;

```tsx
<NotifyModal
  title={'Modal with timeout'}
  message={'Display an alert after 3 second timeout'}
  close={() => setModalVis(false)}
  closeLabel={'close'}
  modalType={'success'}
  timeout={3000}
  callback={alertTimeout}
/>
```

:::info[Omitting the callback function]
If you set a `timeout` you should also set the `callback` function. But there is a case where it is recommended to omit it,
since omitting the `callback` function will result in the `close` function being used as a fallback. \
This is useful for when both functions are identical, as you won't have to provide the same function twice.
:::
