# Button

The Button component is a versatile UI element designed to trigger actions through a click or tap.

## Variants
The button component offers three distinct variants: `filled`, `outlined` and `text`.

<DefaultButtons/>

<details>
  <summary>Expand code</summary>

  ```jsx
  <Button variant={'filled'}>Filled</Button>
  <Button variant={'outlined'}>Outlined</Button>
  <Button variant={'text'}>Text</Button>
  ```
</details>

<Callout type={'info'}>
  Each variant can be converted into a link-button by providing the `href` prop. This allows the button to function as a
  clickable link while maintaining its button style and behavior.
</Callout>

## Filled
Filled buttons are high-emphasis elements, typically used for primary actions within your application.

<Buttons variant={'filled'}/>

<details>
  <summary>Expand code</summary>

  ```jsx
  <Button variant={'filled'}>Filled</Button>
  <Button variant={'filled'} disabled>Disabled</Button>
  <Button variant={'filled'} href={'#filled'}>Link</Button>
  ```
</details>

## Outlined
Outlined buttons are medium-emphasis elements, commonly used for secondary actions within your application.

<Buttons variant={'outlined'}/>

<details>
  <summary>Expand code</summary>

  ```jsx
  <Button variant={'outlined'}>Outlined</Button>
  <Button variant={'outlined'} disabled>Disabled</Button>
  <Button variant={'outlined'} href={'#outlined'}>Link</Button>
  ```
</details>

## Text
Text buttons are low-emphasis elements, generally used for less critical actions or links within your application.

<Buttons variant={'text'}/>

  <Button variant={'text'} disabled>Disabled</Button>

## Color Options
Each button variant offers predefined color options, along with the ability to set a custom color. Below are the available options:

<ColorButtons/>

<details>
  <summary>Expand code</summary>

  ```jsx
  <Button variant={'filled'} color={'info'}>Info</Button>
  <Button variant={'filled'} color={'success'}>Success</Button>
  <Button variant={'filled'} color={'warning'}>Warning</Button>
  <Button variant={'filled'} color={'error'}>Error</Button>
  <Button variant={'filled'} color={'#37edde'}>Custom</Button>
  <Button variant={'filled'} color={'#9437ed'}>Custom</Button>
      
  <Button variant={'outlined'}>Info</Button> // You can omit the color option if you want to use info
  <Button variant={'outlined'} color={'success'}>Success</Button>
  <Button variant={'outlined'} color={'warning'}>Warning</Button>
  <Button variant={'outlined'} color={'error'}>Error</Button>
  <Button variant={'outlined'} color={'#37edde'}>Custom</Button>
  <Button variant={'outlined'} color={'#9437ed'}>Custom</Button>

  <Button variant={'text'}>Info</Button>
  <Button variant={'text'} color={'success'}>Success</Button>
  <Button variant={'text'} color={'warning'}>Warning</Button>
  <Button variant={'text'} color={'error'}>Error</Button>
  <Button variant={'text'} color={'#37edde'}>Custom</Button>
  <Button variant={'text'} color={'#9437ed'}>Custom</Button>
  ```
</details>

The default color option is `info`.

The font color of filled buttons with a custom color is determined based on the contrast ratio, ensuring optimal
readability by dynamically selecting either black or white. This functionality is implemented using the custom hook
[useContrastColor](/docs/hooks/contrast-color).

<Callout type={'warning'}>
  The custom color must be a valid HEX value, either three or six characters long (e.g., #FFF or #FFFFFF).
</Callout>
